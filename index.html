Quick retrospective

  1. CSS was pushed by Microsoft (cool story about it)
  2. Support for media queries, imports (ability to do several files)
  3. Methodologies – BEM, SMACSS, OOCSS, ACSS, etc
  4. Preprocessors – ability to reuse and write more sane code
  5. CSS modules (in some way or another)
  6. Inline CSS in JS; CSS in JS

We might end up with another LISP in browser for styling

Main problems of CSS

  1. Specifity
  2. Fragile code
  3. Hard to organize files
  4. Verbosity (solved by preprocessors)
  5. No possibility to share variables with application code

All methodologies fight with it (not very successfully)

CSS modules

  1. All about specifity
  2. Create unique styles for you behind the scenes
  3. Allow to require class names from different files
  4. We don't lose anything comparing to CSS, but we can use the same title, content, image class names everywhere
  5. But! Same problems as CSS – we can't share variables.
  6. The main problem is no theming and sharing

    7. Almost ideal for standalone projects

Our requirements

  1. Theming (dynamic theming is desired)
  2. Independent components (packages) should be easily styled and use theme
  3. No specifity hell
  4. Full sharing of variables with application code
  5. Would be nice to have:
    1. CSS pseudoclasses
    2. Actual classnames with some semantics
    3. 1 level inheritance (my personal preference)

Radium and inline-css

  1. Describe all css in js
  2. No "classes" – no inheritance, even in theory
  3. Cumbersome pseudoclasses (implemented via React state of HOC)
  4. Manual styles merge (via objects)
  5. Theming out of the box; but to achieve our requirements, we have to add parent React class and follow few conventions
  6. Non-reliable hot reload (based on our experience)

CSS in JS

  1. This approach creates actual stylesheet and put new classes there
  2. Hot reload (in theory) is much more reliable, because part of the job is done via browser – need to be investigated
  3. Pseudoclasses are native, but no inheritance
  4. Theming is pretty easy
  5. Examples:
    1. Styled components
    2. Aphrodite
    3. Glamor

Styled Components

  1. The most "hip" technology, has the biggest community power
  2. Also, it is the youngest technology, so errors are more than possible
  3. Uses template literals, therefore quite unfamiliar syntax
  4. We would have to change our style
